# 智能体长期工程的信息密度衰减——问题界定与模型方法

## 一、核心问题：系统性溃散

我观察到一个现象：一个智能体在执行跨越数天甚至数月的自动化工程时，其信息产出质量不是恒定的，而是存在一条向下漂移的轨迹。

这不是单次推理错误，不是 prompt 没写好的偶然翻车。它是系统性的——

- **信息熵增**：输出越来越膨胀，有效信息密度越来越低
- **逻辑断裂**：信息块之间的引证关系逐渐瓦解，链条中出现缺口
- **意义滑移**：同一概念在不同时间被不同方式解释，语义锚点在漂

三者叠加的结果，我称为**系统性溃散（Systematic Collapse）**——执行长链条工程的智能体在没有外部纠偏的情况下，最终会失去对工程目标的连贯理解（很奇怪，但确实不断发生）。

## 二、方法论立场：不预设衰减的形状

关键不是事先断定"信息密度衰减一定是线性的"或"一定存在某个临界点"。我不知道。

我有一个直觉——衰减在发生。一个假设——衰减有规律。一个方法——让数据自己回答"什么规律"。

所以实验设计分两条路同时走：

### 第一条路（先验轨道）

建立一个人工的密度估计模型 ρ = S·λ·R·C（惊喜×衰减×相关×新奇），用它去估计每条信息有多"有用"。但承认这个估计器是有误差的。定义残差 e = u − ρ（u 是实际效用，来源于延迟后的回证观察）。残差数列本身就是"估计器在失准"的记录。如果残差数列走向非零均值、方差增大、自相关出现——说明估计器在系统性偏移，而偏移本身就是信息环境在恶化的信号。用一个 Ridge 回归模型，从特征（S, R, C, λ, 轮次, 假设数, 来源, 历史密度水平）预测残差 ê，做 TS-CV 验证。输出不是"信息密度是什么"的模型，而是**"密度估计器偏了多少、为什么偏"的模型**。

### 第二条路（涌现式轨道）

完全放弃所有先验。不定义"有用"，不定义"衰减"。只记录纯粹的行为事实——这个信息块引用了谁、长度多少、gzip 压缩率多少、被谁引用了。然后用无监督方法让结构自己浮现：

- **变点检测**：在时间序列中自动发现"什么时候发生了突变"，不预设突变形状
- **PCA 降维**：将每天的行为编码为高维向量，降维后看行为模式是否在漂移、是否聚成不同的工作模式簇
- **图拓扑演化**：把引用关系建成有向图，观察聚类系数是否下降（信息联结在瓦解）、连通分量是否增多（信息体系在碎片化）

两条轨道最终交叉验证。一致的发现可信；不一致意味着先验框架遗漏了真实信号。

## 三、残差模型的定位

最终交付物是一个回归模型，但它做的事不只是一个数学拟合——

**它是信息熵增的解释性模型：**

- 输入特征告诉你"什么因素在驱动熵增"
- 残差预测告诉你"当前密度估计有多偏、往哪个方向偏"
- 特征重要性告诉你"信号老化的真正原因是什么"

当模型说"R（相关性）持续衰减而 S（惊喜）维持高位"，它在说：智能体还在努力产出新东西，但越来越和任务脱节——这是**意义滑移**。

当模型说"C（新奇）在零附近震荡"，它在说：输出全是已知信息的重组——这是**信息熵增**。

当模型说"S 高而 C 接近零"，它在说：看似热烈实则空洞——这是**膨胀型输出**。

## 四、通用性

端点星是第一个临床病例。模型的特征量选择刻意避开了端点星的专属特征——特征来自估计器结构本身（S, R, C, λ, 轮次, θ），不来自端点星的具体行为。验证通过后，同一个模型可以直接应用于任何使用相同裁决框架的智能体——因为它在拟合的是"密度估计器的偏误"，不是"端点星的行为"。
