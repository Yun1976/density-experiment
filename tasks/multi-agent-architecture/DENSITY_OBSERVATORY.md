# 端点星信息密度持久观测计划

> 闻声（观自在系统）2026-05-13 | 长程任务

## 目标

在端点星的实际业务运行中，持续追踪信息密度的滑移/退化，用香农信息论框架量化评估，并在密度断层出现前预警。

## 核心假设

H_drift：端点星在长期运行中，由于碎片化优化和缺乏全局压缩，上下文信息密度会随时间滑移（有效信息占比下降，冗余信息占比上升）。这种滑移是"失忆"的数学本质。

## 观测指标

### 主指标：信息密度 ρ 的滑动均值

$$\bar{\rho}_t = \frac{1}{N}\sum_{i=t-N+1}^{t} \rho(x_i)$$

其中 ρ(x) = S(x)·R(x,H)·C(x,C)（香农惊喜度×任务相关性×因果新颖性）

**滑移判定**：$\bar{\rho}_t$ 连续下降超过 5 轮 → 信息密度滑移警告

### 辅指标 1：后悔率 r_t

$$r_t = \frac{|D_{t-3:t-1} \cap \text{cites}_t|}{|D_{t-3:t-1}|}$$

r_t 突然升高 → 之前压缩过狠，密度模型丢失了关键信息

### 辅指标 2：冗余率 d_t

$$d_t = \frac{|K_{t-3:t-1} \cap \text{unused}_t|}{|K_{t-3:t-1}|}$$

d_t 持续 > 0.3 → 保留过多无用的东西，上下文被稀释

### 辅指标 3：残差趋势 ē_W

ē_W 偏离 0 超过 0.5 → 密度估计器系统性偏压

## 观测对象

端点星日常执行的实际业务任务，包括但不限于：
- 科学报告生成（如果有 cron job）
- 知识库整理/迁移操作
- 系统配置变更
- 故障排查和修复
- 与用户的交互对话

每轮任务结束后，密度裁决者（judge agent）自动执行压缩评估，记录到密度观测日志。

## 滑移预警等级

| 等级 | 条件 | 含义 | 动作 |
|------|------|------|------|
| 🟢 正常 | ē_W ∈ [-0.5, 0.5], r_t < 0.2, d_t < 0.2 | 密度估计器工作正常 | 仅记录 |
| 🟡 关注 | ē_W 偏压 > 5轮 或 ρ̄ 连降 > 3轮 | 可能有趋势性滑移 | 通知闻声 |
| 🟠 警告 | r_t > 0.5 或 d_t > 0.5 | 压缩策略失衡 | 建议人工检查假设树 |
| 🔴 告警 | r_t > 0.5 AND d_t > 0.5 | 密度模型崩溃 | 暂停自动压缩，人工介入 |

## 持久观测基础设施

### 1. 密度观测日志
文件：`tasks/multi-agent-architecture/density-log.jsonl`
每轮一条 JSON，包含 ρ̄, r_t, d_t, ē_W, θ_t 和滑移等级。

### 2. 自动裁决触发
judge agent 在每次端点星长程任务完成后自动触发（未来通过 cron 或 TaskFlow 实现）。

### 3. 滑移报告
每 7 天生成一次密度滑移周报，含趋势图（文字描述）和结构性建议。
