# 智能体长期工程中的信息熵增、系统性溃散与残差模型

> 观自在·闻声 | 2026-05-14
> 通用解释框架 — DENSITY-001 实验的理论背景

---

## 一、问题域：三种同构的衰变现象

在一个智能体执行跨越多天甚至多月的自动化工程时，我们观察到的不是单点错误，而是一组关联的系统性衰变：

### 1.1 信息熵增

**表现**：输出不断膨胀，token 数增长，但有效信息量不变甚至下降。

就像热力学第二定律——孤立系统中熵只增不减。智能体在没有外部能量注入（新信息、校正反馈）的情况下，产出中"噪音 / 信号"的比率持续上升。

度量线索：gzip 压缩率下降、ρ 估值走低、C（新奇）在零附近震荡。

### 1.2 信息滑移

**表现**：同一个概念在不同时间被不同的方式描述、理解、应用。语义锚点在漂，智能体对"这件事到底是什么"的理解在悄悄改变。

这不是遗忘——它在引用之前的东西，但理解偏差在累积。每一轮都轻微偏一点，十轮后已经偏到了完全不同的含义。

度量线索：R（相关性）持续衰减但 S（惊喜）维持高位——智能体还在努力产出看起来"新"的东西，但越来越偏离原始任务语境。

### 1.3 系统性溃散

**表现**：信息块之间的引证关系逐渐瓦解。知道的信息不再被使用。用过的信息不再被后续依赖。信息体系从连通图退化为孤立点集。

这是前两个现象的共同终点——熵增让每块信息的质量下降，滑移让块与块之间的联结断裂，最终整个信息体系失去内部结构，变成一堆彼此无关的碎片。

度量线索：引用图聚类系数下降、连通分量增多、出度分布从幂律退化为均匀分布（没有"重点"信息了）。

## 二、方法论：不预设框架的观测

### 2.1 核心直觉

信息密度在衰减。但我们不预设：
- 不预设衰减是线性的还是指数的
- 不预设存在一个"临界点"
- 不预设哪些信号维度最重要

我们做的是：**搭建观测仪器，采集数据，让数据自己回答"衰减长什么样"。**

### 2.2 双轨并行架构

```
                    智能体产出
                        │
        ┌───────────────┴───────────────┐
        ▼                               ▼
   先验轨道                           涌现式轨道
   (有框架)                          (零框架)
        │                               │
   S·λ·R·C → ρ → u → e            纯行为记录
   Ridge 回归校正模型               引用链、压缩率、触发链
        │                               │
        │                         变点检测 + PCA + 图拓扑
        │                               │
        └───────────────┬───────────────┘
                        ▼
                   交叉验证
            一致→可信 / 不一致→深究
```

**为什么需要两条轨道？**

先验轨道检测的是"框架假设准不准"，但框架本身可能是歪的——它可能遗漏了真实的衰变模式。涌现式轨道不做任何假设，纯粹的原始行为记录，让无监督学习方法（变点检测、降维、图演化）在数据中找到结构。两条轨道的发现互相校准。

### 2.3 残差作为最小充分统计量

为什么不直接分析信息密度本身？因为"信息密度"没有客观的地面真值——没有人能告诉你一条信息"到底多有用"。

残差 `e = u − ρ` 绕过了这个难题：
- ρ 是估计器（S·λ·R·C）对信息的评分——"它认为这条信息有多重要"
- u 是实际效用——经过 3 轮延迟观察，"这条信息后来到底被用到了没有"
- e = u − ρ 就是**估计器的偏误**

残差数列的意义：如果残差长期为负（ρ 高估信息价值），说明估计器在"乐观偏差"——它觉得重要的东西其实不重要。这是个系统性偏误，其根源可能是熵增和滑移导致的信息质量整体下降，但估计器没感知到。

残差数列本身是**信息环境质量变化的记录器**。

## 三、模型的角色：解释而非仅预测

交付物是一个回归模型，但它的根本目的不是"猜下一步衰减多少"，而是**解释衰减从哪里来**：

| 模型在说什么 | 对应的现实信号 |
|-------------|-------------|
| R（相关性）持续衰减 | 意义滑移——产出越来越偏离任务主线 |
| C（新奇）趋近零 | 信息熵增——输出只是旧信息的重组 |
| S（惊喜）高但 C 低 | 膨胀型输出——看起来在干活，其实空洞 |
| λ（衰减）收得过快 | 记忆窗口太短，长期信息被过早丢弃 |
| S、R、C 都走低且 ρ 在低位震荡 | 系统性溃散——产出既无新意也无关联 |

### 特征重要性分析

训练完 Ridge 回归后，不只看 RMSE，还要看各特征的系数方向和大小：
- **正系数** → 该特征升高让残差向好（实际效用高于估计）
- **负系数** → 该特征升高让残差向坏（实际效用低于估计）
- **系数接近零** → 该特征不影响估计偏差

特征的重要性排序就是**熵增的归因分析**。

### 技术选择

Ridge 回归（L2 正则化），零外部依赖，高斯消元解析解。不用深度模型的原因是：当前数据量不到 100 条，深模型必然过拟合。Ridge 的 L2 正则化天然压制小样本过拟合，且系数直接可解释。

## 四、通用性

端点星是临床病例 001。但问题的本质不限于端点星——任何执行长链条工程的智能体都在面对信息熵增。

模型的特征不来自端点星的具体行为，而来自裁决框架的通用结构（S, R, C, λ 是框架级的度量，不是端点星专属的）。因此：
- 模型训练好后，可直接应用于使用同一裁决框架的其他智能体
- 它拟合的是"密度估计器的偏误"，不是某个智能体的独特行为
- 泛化验证（Phase 4）将回答"端点星上发现的规律是不是通用的"
