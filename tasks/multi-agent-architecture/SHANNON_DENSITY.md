## 零、问题陈述

长程任务中，Agent 上下文窗口有限（~128K tokens），但多轮交互产生的上下文持续增长。
核心问题：**如何在每一轮压缩上下文中，最大化保留对完成任务有因果效用的信息？**

这不是一个工程优化问题。这是一个信息论问题。

---

## 一、理论基础

### 1.1 香农三定理与长程任务的对偶

| 香农定理 | 在长程任务中的对偶 |
|----------|-------------------|
| **信源编码定理** | 上下文压缩 = 用最少 tap token 编码"任务关键信息" |
| **信道编码定理** | 上下文窗口 = 有限容量的噪声信道 |
| **率失真定理** | 压缩必然损失信息，需要最小化损失对任务的影响 |

### 1.2 核心公式：信息密度算子

对于上下文中的一个信息块 x，定义其**香农密度**为：

$$\rho(x) = I(x; \mathcal{T}) = H(\mathcal{T}) - H(\mathcal{T}|x)$$

其中：
- $\mathcal{T}$ = 任务目标随机变量（成功 / 失败 / 分支路径）
- $H(\mathcal{T})$ = 任务目标的不确定性（先验熵）
- $H(\mathcal{T}|x)$ = 知道 x 后任务目标剩余的不确定性
- $I(x; \mathcal{T})$ = x 与任务目标之间的**互信息**

**直觉**：信息密度 = "知道 x 后，任务该往哪走的困惑减少了多少 bit"

这是香农互信息的直接应用——不依赖任何启发式。

## 二、可操作的算法

### 步骤 0：定义任务假设树 H

假设树是任务的"贝叶斯先验"——决定什么事是意料之中，什么事是意外。

```
H = {h₁, h₂, ..., hₙ}
每个 hᵢ 是一个关于任务的假设：
  - hᵢ: "应该先做 A 再做 B"
  - hᵢ: "X 是 Y 的依赖"
  - hᵢ: "当前路径可以达成目标"
```

假设树质量决定密度评估质量。假设树空洞 → 密度评估失准 → 压缩丢错东西。

### 步骤 1：估计先验概率 P(x)

**目标**：给定当前假设树和任务历史，x 出现的概率是多少？

#### 方法：Bayesian 先验估计

$$P(x) = \sum_{h \in \mathcal{H}} P(x|h) \cdot P(h)$$

其中：
- $P(h)$ = 假设 h 的当前置信度（初始 0.5，随证据累积更新）
- $P(x|h)$ = 在假设 h 为真的条件下，观察到 x 的概率

#### 工程近似表

| x 的类别 | P(x) | S(x) = -log₂P(x) | 示例 |
|----------|------|------------------|------|
| 预期工具输出（返回已知值） | 0.95 | 0.07 | `node --version` 返回 v22 |
| 预期工具输出（返回新值，格式已知） | 0.85 | 0.23 | `ls dir/` 列出预期目录结构 |
| 常规推理路径 | 0.70 | 0.51 | "先读文件，再修改，再验证" |
| 意外但合理的发现 | 0.20 | 2.32 | "配置文件中发现未知字段" |
| 与某个假设矛盾的证据 | 0.08 | 3.64 | "预期网关在线，实际 502" |
| 推翻已验证假设 | 0.02 | 5.64 | "之前确认的 API 端点已下线" |
| 动摇了任务命题本身 | 0.001 | 9.97 | "政策变更导致整个项目不可行" |

### 步骤 2：估计条件熵 H(T|x)

**关键问题**：知道 x 之后，还有多少关于任务走向的不确定性？

#### 方法：贝叶斯更新 + 熵计算

1. **知道 x 后更新假设置信度**（贝叶斯更新）：

$$P(h|x) = \frac{P(x|h) \cdot P(h)}{P(x)}$$

2. **计算条件熵**：

$$H(\mathcal{T}|x) = -\sum_{h \in \mathcal{H}} P(h|x) \cdot \log_2 P(h|x)$$

3. **计算互信息（最终密度）**：

$$\rho(x) = H(\mathcal{T}) - H(\mathcal{T}|x)$$

其中 $H(\mathcal{T}) = -\sum_{h} P(h) \cdot \log_2 P(h)$ 是先验任务不确定性。

#### 工程上不可直接计算时

在实际 Agent 上下文中，我们无法精确求解 P(x|h)。**使用估计器**：

##### 估计器 A：语义空间投影

$$\text{sim}(x, h) = \text{cosine}(E(x), E(h))$$

其中 E(·) 是嵌入向量（可以是 LLM embedding 或简化为关键词交集/因果匹配）。

**修正版（v1.1，Cycle 3 裁决后修正）**：不取所有假设的 max，仅取**当前活跃假设**的相关性。

$$R(x, \mathcal{H}_{\text{active}}) = \max_{h \in \mathcal{H}_{\text{active}}} \text{sim}(x, h)$$

其中 $\mathcal{H}_{\text{active}}$ = 当前轮次正在验证/关注的假设子集。

**修正理由**：原 max 策略导致"假设树越宽泛，R 虚高"——非活跃假设即使匹配也不该贡献相关性。Cycle 3 实证：残差 ē_W 连续两轮为负（ŷ 系统偏高），部分归因于 R 的虚高估计。

若 $\mathcal{H}_{\text{active}}$ 为空（首轮），回退到全部假设的 max。

##### 估计器 B：因果链路检测

检查 x 是否在任务因果图中提供了新信息：

```python
def causal_impact(x, causal_graph):
    """
    x 在任务因果图中有影响当：
    1. x 填补了某个"未知"节点
    2. x 改变了某条边的方向或权重
    3. x 引入了新的因果节点
    """

    # 距离近且有新信息 → 高影响
```

### 步骤 3：计算香农密度 ρ(x)（混合估计器）

**精确估计（当假设树充分时）：**

$$\rho(x) = H(\mathcal{T}) - H(\mathcal{T}|x)$$

**工程估计（当前实现，v1.2）：**

$$\rho_t(x) = S(x) \cdot \lambda(t) \cdot R(x, \mathcal{H}_{\text{active}}) \cdot C(x, \mathcal{C})$$

其中 λ(t) 是 S 值衰减因子（见 2.1.1 节）。

新增项（v1.1）：
- $C(x, \mathcal{C}) \in [0,1]$ = **因果新颖性**：x 是否为任务因果图 $\mathcal{C}$ 引入了新的信息

三个因子的直觉：
| 因子 | 含义 | 低值 | 高值 |
|------|------|------|------|
| S(x) = -log₂P(x) | 香农惊喜度 | 可预测的 | 意料之外的 |
| R(x, H) | 最大相关性 | 与任务无关 | 与核心假设紧密相关 |
| C(x, C) | 因果新颖性 | 已有信息重复 | 新因果链/新依赖/新约束 |

**三者相乘** → 只有 "既意外、又相关、且新" 的信息才有高密度。

### 2.1.1 S 值衰减因子 λ(t) — v1.2 新增

**动机**：香农惊喜度 S(x) = -log₂P(x) 严格衡量在**首次观察时刻**的意外程度。但信息一旦被认知并写入 TASK.md，"惊喜"就不再生效——已知信息不再是惊喜。Cycle 4 实证：A块（自我反膨胀）ρ=4.82，S=5.64，但后续 u=1（仅被引用而非产生对应5.64级别的决策影响），e=-3.82的高估揭示"极高S≠极高长期效用"。

**衰减函数**：

$$\lambda(t) = \max(0.3, \; 1 - 0.2 \cdot (t - t_{\text{first\_seen}}))$$

其中：
- t = 当前裁决轮次
- t_first_seen = 信息块首次被观察到的轮次

| 轮次差 (t - t_first_seen) | λ(t) | 含义 |
|--------------------------|------|------|
| 0（首轮） | 1.0 | 惊喜全效 |
| 1 | 0.8 | 半衰开始 |
| 2 | 0.6 | 大部分已知 |
| 3+（≥3） | 0.3（地板） | 最少保留权重 |

**设计原则**：
- 地板 λ=0.3 而非 0 → 即使完全已知，信息仍有剩余因果价值（假设树更新、历史对比基线）
- 衰减仅作用于 S 值 → R 和 C 不衰减（相关性和因果新颖性跨轮持久）
- 衰减在 ρ 计算时生效 → 旧信息块在后续裁决中的 ρ 自然下降，不占用保留配额

**对再评估的影响**：
- 首次裁决：ρ = S·1.0·R·C（无衰减）
- 第2次对同一信息块再评估（下一轮后悔检查）：ρ = S·0.8·R·C
- 第4轮后：ρ = S·0.3·R·C → 可能跌穿 θ → 被自然压缩或丢弃

**实证验证**：A块若在 Cycle 3 首评后于 Cycle 4 重评：
- 原 ρ = 5.64 × 0.95 × 0.90 = 4.82
- 重评 ρ = 5.64 × 0.8 × 0.95 × 0.90 = 3.86 → 仍在保留区但更接近真实 u 值

### 2.1.2 保留项定期重评机制 — v1.4 新增

**动机**（Cycle 7 实证）：D块（压缩后三文档状态，C3 ρ=2.32）在 C4-C7 连续4轮未被引用，但其 ρ 从未被降低——因为 S 衰减因子 λ(t) 仅在信息块**被显式重新评分**时生效。已被保留的信息块如果不被重评，其 ρ 永远停留于首轮估值，θ 上升无法自动淘汰旧保留项。

**机制**：每 K=3 轮对全部保留项执行一次重评。

$$\rho_t^{\text{reeval}}(x) = S(x) \cdot \lambda(t) \cdot R(x, \mathcal{H}_{\text{active}}) \cdot C(x, \mathcal{C})$$

其中 λ(t) 基于当前轮次与首次观察轮次的差值计算。

**重评结果处理**：
- ρ_reeval > θ_t → 继续保留（但用新 ρ 替换旧 ρ）
- θ_t × 0.5 < ρ_reeval ≤ θ_t → 降级为压缩摘要
- ρ_reeval ≤ θ_t × 0.5 → 移入低密度候选区（W=3 后清除）

**重评周期**：每 K=3 轮执行一次（与冗余窗口对齐）。重评本身不消耗额外工具调用——在冗余检查步骤中同步完成。

**首个重评轮次**：C9（C1-C6 保留项的首个重评周期）。

## 三、完整的压缩-恢复算法

### 输入

- 当前上下文窗口 C_t（tokens）
- 假设树 H
- 任务因果图 C
- 当前阈值 θ_t

### 主循环伪代码

```
Algorithm SHANNON_COMPRESS(C_t, H, C, θ_t):
  Input:  current context window, hypothesis tree, causal graph, threshold
  Output: compressed context C_{t+1}, updated H, updated C, updated θ

  # Phase 1: 分块
  blocks = segment_context(C_t)  # 按语义边界切分成信息块

  # Phase 2: 逐块评分
  scores = []
  for each block x in blocks:
    surprise = -log2(estimate_prior_probability(x, H))
    relevance = active_hypothesis_similarity(x, H)  # v1.1: only active hypotheses
    novelty = causal_novelty(x, C)    # does x add new info to causal graph?
    decay = s_decay_factor(x, current_round)  # v1.2: S衰减
    
    ρ = surprise * decay * relevance * novelty
    scores.append((x, ρ))

  # Phase 3: 保留/压缩/丢弃
  retained = []
  compressed_segments = []
  dropped = []

  for (x, ρ) in scores:
    if ρ > θ_t:
      retained.append((x, ρ))          # keep as-is
    elif ρ > θ_t * 0.5:
      compressed_segments.append(summarize(x))  # compress to 1-2 sentences
    else:
      dropped.append((x, ρ))           # discard

  # Phase 4: 构造压缩上下文
  C_{t+1} = assemble(retained, compressed_segments)

  # Phase 5: 更新因果图和假设树
  H = bayesian_update(H, retained)     # 更新假设置信度
  C = extend_causal_graph(C, retained) # 加入新因果链

  # Phase 6: 记录裁决结果（为下一轮的后悔/冗余做准备）
  log_density_decisions(retained, dropped, compressed_segments)

  return C_{t+1}, H, C
```

### 阈值自适应校准

```
Algorithm CALIBRATE_THRESHOLD(r_t, d_t, θ_t):
  Input:  regret rate, redundancy rate, current threshold
  Output: new threshold θ_{t+1}

  # α: 后悔学习率 (0.15)
  # β: 冗余学习率 (0.10)
  
  θ_{t+1} = θ_t * (1 - α * r_t + β * d_t)

  # 硬边界
  θ_{t+1} = clamp(θ_{t+1}, θ_min=0.2, θ_max=5.0)

  return θ_{t+1}
```

## 四、残差数列 — 后悔值的科学理论基础

### 4.0.1 定义

密度评估的**残差**：第 t 轮估计值与实际效用之间的偏差。

$$e_t(x) = u_t(x) - \rho_t(x)$$

其中：
- $u_t(x)$ = x 的实际效用（后续被证实的互信息，$\in [0, +\infty)$）
- $\rho_t(x)$ = 第 t 轮估计的密度值
- $e_t(x) \in (-\infty, +\infty)$ = 评估误差
  - $e_t > 0$ → **低估**了 x 的价值（以为没用，实际有用）
  - $e_t < 0$ → **高估**了 x 的价值（以为有用，实际没用）
  - $e_t \approx 0$ → 准确

### 4.0.2 残差数列与后悔的精确关系

后悔值 $r_t$ 在残差框架下的严格定义：

$$\boxed{r_t = \frac{\sum_{x \in D_{t-1}} \max(0, u_t(x))}{|D_{t-1}|}}$$

用残差表示：

$$r_t = \frac{|\{x \in D_{t-1} : e_{t-1}(x) < -|\rho_t(x)|\}|}{|D_{t-1}|}$$

**翻译**：残差为大负值（严重高估为零，丢弃了实际很重要的东西）→ 后悔。

### 4.0.3 残差数列的统计性质

#### 期望 — 无偏性
若密度估计器长期无系统性偏见：
$$E[e_t] = 0$$
$|\bar{e}| > 0.5$ 连续 5 轮 → 估计器偏压警告。

#### 方差 — 估计器精度
$$\sigma_e^2 = \frac{1}{T}\sum_{t=1}^{T} e_t^2$$
- $\sigma_e < 0.5$ → 密度估计精准，后悔值可信
- $\sigma_e > 1.5$ → 密度估计不稳定，后悔值含高噪声

**这是元评估指标：衡量密度算法本身的质量。**

#### 自相关 — 趋势检测
$$ACF(k) = \frac{\sum_{t=k+1}^{T} (e_t - \bar{e})(e_{t-k} - \bar{e})}{\sum_{t=1}^{T} (e_t - \bar{e})^2}$$
- $ACF(1) > 0.5$ → 残差有持续性（连续低估/高估），θ 调整过慢
- $ACF(1) \approx 0$ → 白噪声，θ 校准合适
- $ACF(1) < -0.3$ → 过度校正，θ 振荡

### 4.0.4 残差驱动的阈值校准（升级版）

原始公式：$\theta_{t+1} = \theta_t \cdot (1 - \alpha r_t + \beta d_t)$

**升级后加入残差趋势项**：

$$\boxed{\theta_{t+1} = \theta_t \cdot (1 - \alpha r_t + \beta d_t - \gamma \cdot \bar{e}_t^{\text{window}})}$$

新增：
- $\bar{e}_t^{\text{window}}$ = 最近 W 轮的残差滑动均值
- $\gamma$ = 残差修正学习率（初始 0.05）
- $\bar{e}_t > 0$ → 长期高估密度 → 降低 θ（更多保留）
- $\bar{e}_t < 0$ → 长期低估密度 → 升高 θ（更多丢弃）

**意义**：这个项让算法不仅能响应后悔/冗余，还能响应**系统性偏见**。

### 4.0.5 残差预警信号

| 信号 | 条件 | 含义 | 动作 |
|------|------|------|------|
| 漂移 | $|\bar{e}_{t}^{W}| > 2\sigma_e^{\text{hist}}$ | 密度估计器系统漂移 | 检查假设树是否过时 |
| 爆炸 | $|e_t| > 3\sigma_e^{\text{hist}}$ | 单个异常判定 | 标记该信息块，重新评估 |
| 偏压 | $\text{sign}(\bar{e}_{t}^{W})$ 持续 > 5 轮 | 方向性估计偏差 | 人工检查估计器 |

### 4.0.6 残差收敛定理

**定理（残差收敛准则）**：当且仅当残差数列 $\{e_t\}$ 满足：
1. $E[e_t] \to 0$（无偏）
2. $\text{Var}[e_t] \to \sigma^2$ 有界（方差收敛）
3. $ACF(k) \to 0$ 对所有 $k \geq 1$（不相关/白噪声）

则 $\theta_t$ 收敛到最优阈值 $\theta^*$。

**逆命题**：若残差长期有偏，则任何后悔/冗余校准都在错误方向上优化。

## 四、后悔值 r_t — 香农框架下的严格定义

### 4.1 形式化定义

在香农信息论下，后悔 = **丢弃信息中被后续证明有互信息的比例**。

$$r_t = \frac{\sum_{x \in D_{t-1}} I(x; \mathcal{T}) \cdot \mathbb{1}[x \text{ was needed at round } t]}{\sum_{x \in D_{t-1}} I(x; \mathcal{T})}$$

其中：
- $D_{t-1}$ = 第 t-1 轮丢弃的信息块集合
- $I(x; \mathcal{T})$ = x 与任务目标的互信息（即丢弃时认为 x 的密度很低）
- $\mathbb{1}[\ldots]$ = 指示函数，x 在第 t 轮被引用/依赖则为 1

**直觉**：后悔不是简单地数"丢错了几条"——而是用互信息加权。"错误地丢掉了高互信息的东西"比"错误地丢掉了低互信息的东西"后悔更大。

### 4.2 三信号分解

#### 信号 1：引用后悔 r_cite

第 t 轮中，agent 引用了"之前看到/发现/确认的 X"，但 X 已被丢弃。

```
扫描第 t 轮输出：
  - "根据之前..." / "之前发现的..." 
  - "记得..." / "刚才..."
  - 对历史时间戳/路径/数值的引用

若被引用内容 ∈ D_{t-1} → 后悔
```

$$r_{\text{cite}} = \frac{|D_{t-1} \cap \text{ cites}_{t} |}{\max(|D_{t-1}|, 1)}$$

#### 信号 2：重复发现后悔 r_repeat

第 t 轮执行了与第 t-1 轮相同（或等效）的操作，证明丢弃的信息是有用的。

```
比较工具调用参数：
  - 相同的命令 + 相同的参数
  - 相同的搜索查询
  - 相同的文件读取

若重复操作等价于某个丢弃的工具输出 → 后悔
```

$$r_{\text{repeat}} = \frac{|\text{重复操作且等价于 } D_{t-1} \text{ 项}|}{\max(|D_{t-1}|, 1)}$$

#### 信号 3：信息缺口后悔 r_gap

Agent 在第 t 轮因缺少上下文而做出了错误推理，后来发现缺失的信息在 D_{t-1} 中。

```
检测推理自纠错：
  - Agent 说了 A，然后改成 B
  - 纠错原因是"忘了 X"
  - X ∈ D_{t-1} → 信息缺口后悔
```

$$r_{\text{gap}} = \frac{|\text{信息缺口导致的自纠错次数}|}{|\text{总推理步数}|}$$

### 4.3 加权综合

$$r_t = 0.5 \cdot r_{\text{cite}} + 0.3 \cdot r_{\text{repeat}} + 0.2 \cdot r_{\text{gap}}$$

### 4.4 后悔窗口（解决延迟后悔）

后悔可能延迟 1-3 轮才显现。定义后悔窗口 W=3：

$$r_t^{\text{window}} = \frac{\sum_{x \in D_{t-W:t-1}} I(x; \mathcal{T}) \cdot \mathbb{1}[x \text{ needed at round } t]}{\sum_{x \in D_{t-W:t-1}} I(x; \mathcal{T})}$$

### 4.4.1 软后悔 — v1.3 新增

**动机**（Cycle 6 实证）：压缩层（compression layer）可以系统性吸收后悔信号。当被丢弃信息的核心语义通过压缩摘要存活时，它不会触发 r_cite 或 r_repeat——但这不意味着丢弃决策完美。压缩摘要能恢复上下文 ≠ 原始信息不值得保留。

**定义**：

$$r_t^{\text{soft}} = \frac{\sum_{x \in D_{t-1}} I(x; \mathcal{T}) \cdot \mathbb{1}[\text{compress}(x) \text{ cited at round } t]}{\sum_{x \in D_{t-1}} I(x; \mathcal{T})}$$

即：丢弃项中，其压缩摘要被后续引用的比例。

**加权综合（v1.3）**：

$$r_t^{\text{total}} = r_t + 0.3 \cdot r_t^{\text{soft}}$$

软后悔权重 0.3（而非 1.0）因为压缩摘要提供了恢复路径——后悔代价被压缩层降低但未消除。

**压缩层渗透率**（compression penetration rate）：

$$\text{CPR}_t = \frac{|\text{压缩摘要被引用的次数}|}{|\text{总保留引用次数}|}$$

- CPR > 0.5 且 r_t = 0 → 压缩层过度有效，可能掩盖真实后悔 → 触发 θ 降低

### 4.5 初始化的后悔处理

- 前 3 轮：不计算后悔（缺乏足够的丢弃历史），保持 θ = θ₀
- 第 4 轮起：启用后悔窗口 W=3
- 若连续 N=5 轮 r_t = 0 → 主动试探降低 θ 10%（防止阈值过高导致的"安静的灾难"）
- **v1.3 增强**：N=5 触发时，若标准校准方向（基于 d_t + ē_W）与降低 θ 方向相反 → 静默协议优先，因为长期零后悔是小概率信号，优先于偏压/冗余信号

## 四点五、膨胀型输出 — 密度滑移的数学签名

### 4.5.1 定义

膨胀型输出 = 高 token 消耗但信息增量为零的输出。在香农框架下：

$$\text{inflated}(x) \iff S(x) < 0.1 \land C(x) < 0.1$$

即：既不可预测（低惊喜度），又不带来新因果信息（低新颖性）。即使 R 尚可，S 和 C 双双趋零意味着 ρ → 0。

### 4.5.2 典型场景

| 场景 | S(x) | C(x) | 说明 |
|------|------|------|------|
| 连续 N 天相同的巡检报告 | →0 | →0 | 检测工具输出稳定，增量信息为零 |
| 重复的工具调用输出 | →0 | →0 | 预期内的输出，无新信息 |
| 格式化模板填充（无实质变化） | →0 | →0 | 仅有模板字段替换，无因果新信息 |
| "看起来在工作"的日志 | →0 | →0 | 低惊喜、低新颖、只消耗 token |

### 4.5.3 滑移检测

$$\text{drift\_warning} \iff \text{连续 K 轮存在 } \rho(x) < 0.01 \text{ 的输出且未被系统识别}$$

K=2（初始）。连续 2 轮膨胀 → 🟡 关注；连续 3 轮 → 🟠 警告。

### 4.5.4 对抗膨胀的系统设计原则

1. **检测必须伴随修复**：只检测不修复的脚本 = 恒定熵产生器
2. **重复输出是失效模式**：如果连续 2 轮输出相同，不应产生新报告
3. **自反膨胀优于外部压缩**（H6 假设，置信度 0.6）：系统自发识别并压缩冗余比外部裁决更有效

## 五、冗余值 d_t — 香农框架下的定义

### 5.1 形式化定义

冗余 = **保留的信息中，在过去 K 轮内从未被引用/利用的比例**。

$$d_t = \frac{|K_{t-K:t-1} \cap \text{unused}_t|}{|K_{t-K:t-1}|}$$

其中：
- $K_{t-K:t-1}$ = 过去 K 轮保留的信息块集合
- $\text{unused}_t$ = 第 t 轮从未被引用/依赖/利用的信息块

### 5.2 冗余窗口 K

K = 3 轮。保留后超过 3 轮未被引用 → 视为冗余。

### 5.3 冗余驱动的阈值校准

冗余率 d_t 是 θ 的"上升信号"：
- d_t 高 → 保留太多没用的东西 → θ 上升（更严格）
- d_t 低 → 保留的东西都有用 → θ 不变

## 六、完整的信息密度算法

### 主算法

```
Algorithm SHANNON_DENSITY_MANAGER(task_context, hypothesis_tree, causal_graph, θ):
  
  # 每轮执行后的压缩-裁决循环
  for each round t in 1..∞:

    # Step 1: 评估当前上下文的信息密度
    blocks = segment(task_context[t])
    scores = []

    for x in blocks:
      S = -log2(prior_probability(x, hypothesis_tree))
      R = max_similarity(x, hypothesis_tree)
      C = causal_novelty(x, causal_graph)
      ρ = S * R * C

      scores.append((x, ρ))

    # Step 2: 保留/压缩/丢弃
    retained_t, compressed_t, dropped_t = classify(scores, θ_t)

    # Step 3: 先执行保留→写回 TASK.md
    update_task_state(retained_t, compressed_t)

    # Step 4: 执行下一轮任务

    # Step 5: 在 t+1 轮评估 t 轮的裁决质量

    # Step 6: 计算后悔值
    r_t = compute_regret(dropped_{t-1}, round_t)

    # Step 7: 计算冗余值
    d_t = compute_redundancy(retained_{t-3:t-1}, round_t)

    # Step 8: 校准阈值
    θ_{t+1} = θ_t * (1 - α*r_t + β*d_t)
    θ_{t+1} = clamp(θ_{t+1}, 0.2, 5.0)

    # Step 9: 记录裁决日志
    log_density_cycle(t, θ_t, r_t, d_t, retained_t, dropped_t)

    # Step 10: 更新假设树和因果图
    hypothesis_tree = bayesian_update(hypothesis_tree, retained_t)
    causal_graph = extend_causal_graph(causal_graph, retained_t)

  return
```

### 收敛性保证

**定理**：在以下条件下，θ 收敛到不动点 θ*：
1. r_t 和 d_t 的估计无偏（即足够长的后悔窗口和经验积累）
2. 任务的信息分布是平稳的（即任务类型不变）

**证明（直觉）**：
- θ 过高 → 丢弃过多 → r_t ↑ → θ ↓（负反馈）
- θ 过低 → 保留过多 → d_t ↑ → θ ↑（正反馈）
- 两个方向的恢复力形成一个稳定吸引子

**收敛速度**：约 O(log(1/ε)) 轮，其中 ε 是目标精度。
（因为 θ 的乘法更新本质上是指数衰减的形式）

## 六点一、N=5 静默灾难与标准校准冲突处理 — v1.3 新增

### 6.1.1 问题

当 r_t = 0 连续 5 轮时，静默协议要求降低 θ 10%，但标准校准可能基于 d_t（冗余率）和 ē_W（残差均值）指向相反方向。Cycle 6 实例：
- 标准校准：ē_W = -0.553（高估偏压 → ↑θ） + d₅ = 0.30（高冗余 → ↑θ）→ θ ↑
- 静默协议：r_t = 0 × 5 → θ ↓

### 6.1.2 冲突处理规则

```
if N=5 SILENT_DISASTER triggered:
    if standard_calibration_direction == DOWN:       # 方向一致
        θ_trial = min(0.90·θ_current, standard_θ)   # 取更激进的下调
    elif standard_calibration_direction == UP:       # 方向冲突
        θ_trial = 0.90·θ_current                     # 静默协议优先
        log("标准校准与静默协议方向冲突，静默协议优先")
        # 理由：长期 r=0 是小概率事件，权重高于偏压/冗余信号
        # 但设置回滚条件：若连续2轮 d_t > 0.3 → 回滚
```

### 6.1.3 试验窗口与回滚

- 试验窗口：N_trial = 3 轮
- 回滚条件：
  1. 连续 2 轮 d_t > 0.3 → θ 回滚至试验前值
  2. r_t 首次 > 0 且 d_t 未超 0.3 → 试验"成功"（阈值降低暴露了真实后悔），进入标准校准
  3. 3 轮后 d_t 稳定在 ≤ 0.25 且 r_t 仍为 0 → 试验"温和成功"（阈值降低无副作用），保持低 θ
