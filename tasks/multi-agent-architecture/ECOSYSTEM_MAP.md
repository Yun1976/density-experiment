## 二、香农信息密度算法在体系中的位置

SHANNON_DENSITY.md 处于第1层和第2层的交界处：

- 它是第1层的**运行算法**——每轮密度裁决、后悔/冗余/残差计算
- 它也是第2层的**理论基础**——为 CONTEXT-COMPRESSOR-DESIGN.md 提供数学正当性

### 2.1 SHANNON_DENSITY 与 CONTEXT-COMPRESSOR 的互补关系

| 维度 | CONTEXT-COMPRESSOR | SHANNON_DENSITY |
|------|-------------------|-----------------|
| 目标 | 上下文长度压缩 | 信息密度最大化 |
| 方法 | 工程5阶段流水线 | 香农互信息评分 |
| 决策逻辑 | 固定阈值（50%触发） | 自适应阈值（后悔/冗余/残差驱动） |
| 输出 | 压缩后的上下文 | 保留/压缩/丢弃三元判定 |
| 理论基础 | 工程经验 | 信息论 |
| 角色 | **怎么压**（执行器） | **压什么**（决策器） |

两者不是竞品，是镜像。压缩器负责工程执行，密度模型负责决策质量。

### 2.2 理论-实践的通约实例

CONTEXT-COMPRESSOR 的第4阶段摘要模板有13个字段，其中"Active Task"被标注为**最重要字段**。

在香农框架下可以严格证明：

$$I(\text{Active Task}; \mathcal{T}) = \max_{x \in \text{context}} I(x; \mathcal{T})$$

因为 Active Task 是当前任务目标的直接编码，其互信息必然在所有上下文信息块中最大。

**CONTEXT-COMPRESSOR 的工程直觉（"Active Task 最重要"）被 SHANNON_DENSITY 的香农互信息严格证明了。** 这是理论与实践的通约。

### 2.3 SHANNON_DENSITY 对 CONTEXT-COMPRESSOR 的改进方向

| CONTEXT-COMPRESSOR 当前 | SHANNON_DENSITY 建议改进 |
|------------------------|-------------------------|
| `threshold_percent=50%` 固定触发 | θ 自适应校准（后悔/冗余/残差驱动） |
| "反弹跳保护"（连续2次<10%节约→跳过） | 膨胀型输出检测 S<0.1∧C<0.1 |
| 摘要模型失败→降级为主模型 | 残差偏压检测（连续5轮ē<0→调整R策略） |
| 13字段结构摘要 | 保留/压缩/丢弃三元判决（ρ>θ保留, ρ∈[0.5θ,θ)压缩, ρ<0.5θ丢弃） |

## 三、ai-agent-incidents 对密度研究的价值

33条教训的记录格式（场景/症状/根因/解决/预防）本身就是**已知因果链**。

在香农密度算法的因果图 C(x, C) 中：
- 每条教训 = 因果图的一个已知节点
- "根因→症状" = 一条边
- "解决→预防" = 另一条边

**33条教训可以作为因果图 C 的种子输入**——不是从零建因果图，而是在已有的因果骨架基础上扩展。

此外，001号事故（14小时全系统停摆）是密度滑移的极端案例：
- 委托任务给子 agent → 子 agent 的工具未测试 → 沉默失败 → 丢失所有中间状态的"信息断裂"
- 从密度角度看：这是一个 ρ→0 的灾难——14小时内系统的有效信息密度归零

# 信息密度理论基础 — 长程任务上下文管理

> 基于香农（Claude Shannon）信息论的核心定理
> 与 CONTEXT-COMPRESSOR-DESIGN.md 构成互补体系

## 核心问题

长程任务中，AI Agent 的上下文窗口有限，但多轮交互产生的上下文持续增长。核心问题：**如何在每一轮压缩中，最大化保留对完成任务有因果效用的信息？**

这不是工程优化问题——这是信息论问题。

## 理论基础：香农互信息

对于上下文中的一个信息块 x，定义其**香农密度**为：

$$\rho(x) = I(x; \mathcal{T}) = H(\mathcal{T}) - H(\mathcal{T}|x)$$

其中：
- $\mathcal{T}$ = 任务目标随机变量
- $H(\mathcal{T})$ = 任务不确定性的先验熵
- $H(\mathcal{T}|x)$ = 知道 x 后任务剩余不确定性
- $I(x; \mathcal{T})$ = 互信息

**直觉**：知道 x 之后，任务该往哪走的困惑减少了多少 bit。

## 工程估计公式

$$\rho(x) = S(x) \cdot R(x, \mathcal{H}_{\text{active}}) \cdot C(x, \mathcal{C})$$

| 因子 | 含义 | 低值 | 高值 |
|------|------|------|------|
| $S(x) = -\log_2 P(x)$ | 香农惊喜度 | 可预测的 | 意料之外的 |
| $R(x, \mathcal{H}_{\text{active}})$ | 与当前活跃假设的相关性 | 无关 | 与核心假设紧密 |
| $C(x, \mathcal{C})$ | 因果新颖性 | 已有信息重复 | 新因果链/新约束 |

## 自适应阈值

$$\theta_{t+1} = \theta_t \cdot (1 - \alpha r_t + \beta d_t - \gamma \cdot \bar{e}_t^{\text{window}})$$

- $r_t$ = 后悔率（丢弃的信息中被后续证明需要的比例）
- $d_t$ = 冗余率（保留的信息中从未被引用的比例）
- $\bar{e}_t$ = 残差滑动均值（估计密度与实际效用的偏差）

## 膨胀型输出检测

$$\text{inflated}(x) \iff S(x) < 0.1 \land C(x) < 0.1$$

连续2轮存在 ρ<0.01 的输出 → 🟡 关注；连续3轮 → 🟠 警告。

例：每日巡检报告连续2天输出相同（41断链0修复）→ ρ=0.001，典型膨胀型输出。

## 与 CONTEXT-COMPRESSOR 的关系

CONTEXT-COMPRESSOR 负责"怎么压"（5阶段工程流水线），密度模型负责"压什么"（保/压/丢三元判定）。两者互补：

- 压缩器压缩后，密度模型的 θ 下降（保留更少、丢弃更多）
- 密度模型发现后悔后，压缩器的触发策略可调整（更保守或更激进）
- 压缩器的13字段摘要模板 = 密度模型"压缩"操作的推荐输出格式

## 收敛性质

残差数列 $\{e_t\}$ 满足无偏、方差有界、不相关时，$\theta_t$ 收敛到最优阈值 $\theta^*$。
