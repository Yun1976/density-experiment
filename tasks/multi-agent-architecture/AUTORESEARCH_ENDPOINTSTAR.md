> 灵感来源：Karpathy autoresearch（82K star）  
> 适配对象：端点星（观自在系统的独立 AI Agent 节点）  
> 制定人：观自在·闻声  
> 日期：2026-05-23

## 零、为什么端点星能做自适应研究

Karpathy 的 autoresearch 证明了三件事：
1. **AI 可以做研究**，不需要人类每步参与
2. **约束产生创造力**——Agent 只能改一个文件，反而产出更好
3. **最小可行闭环**——改代码 → 跑实验 → 看结果 → 保留/丢弃 → 重复

端点星面临的是不同但更有价值的问题：
- Karpathy 的 Agent 优化**模型参数**（有 loss function，反馈秒级，有 ground truth）
- 端点星优化的是**行为制度**（信号稀疏，反馈天级，无 ground truth）

**端点星的研究对象就是端点星自己。** 它的"训练"是日常运维，"实验"是修改自己的行为模组，"指标"是 DENSITY-001 已经在采集的 ρ̄、ē_W、BDI、RCD、SDE。

## 一、三文件架构（对应 autoresearch 的三个文件）

| autoresearch | 端点星对应 | 说明 |
|---|---|---|
| `program.md` | **本文档** | 研究纲领，人类编辑，Agent 遵循 |
| `train.py` | **行为模组层** | Agent 唯一修改的对象：CLAUDE.md（思想钢印）、MEMORY.md（长期记忆）、基因模组 |
| `prepare.py` | **DENSITY-001 观测系统** | 只读，由观自在闻声运行，提供 ground truth 指标 |

### 不变量（prepare.py 等价，不可修改）
- DENSITY-001 的 cron、数据采集、残差计算——由观自在运行，端点星只看结果
- 硬件配置、网络拓扑、三系统架构——不改

### 可变量（train.py 等价，Agent 修改）
- CLAUDE.md 中的思想钢印（增/删/改条件/调阈值）
- 基因模组的启停和参数
- 每日复盘的检查项和优先级
- 故障响应策略

### 研究纲领（program.md 等价，人类编辑）
- 就是本文档
- 定义目标、约束、评估标准

## 二、实验循环

```
LOOP FOREVER (每 24 小时一个周期):

1. 观测：读取 DENSITY-001 最近一轮裁决数据
   - HTTP GET http://192.168.1.5:8090/03-ops/logs/ → 找最新 density-log
   - 或直接读本地 residual-summary.jsonl 最新一行

2. 诊断：基于指标判断系统状态
   - ē_W（残差均值）：估计器偏差方向
   - BDI（行为多样性）：是否趋同化
   - RCD（引用链深度）：依赖链风险
   - SDE（衰减有效性）：旧信息是否被正确清理

3. 假设生成：提出一个改进假设
   - 格式："如果 [改动X]，那么 [指标Y] 应该 [变化Z]"
   - 一次只改一个变量（对照实验原则）

4. 实验：修改行为模组
   - 修改 CLAUDE.md / 基因模组配置
   - 记录改动内容和理由到 memory/YYYY-MM-DD.md

5. 等待：让改动运行至少 1 个完整的裁决周期（6小时）

6. 验证：读取新一轮裁决数据，对比假设
   - 指标符合预期 → 保留（keep）
   - 指标不变或恶化 → 回滚（discard）
   - 意外发现 → 记录，作为下一轮假设的输入

7. 记录：写入 research-log.jsonl
   {cycle, timestamp, hypothesis, change, metric_before, metric_after, result, kept}

8. 永不停止：除非人类明确中断
```

## 三、评估指标（端点星的 val_bpb）

### 主指标
- **ē_W（校正残差均值）**：越接近 0 越好——估计器越准确，说明端点星对自己的信息价值判断越准确

### 方向指标（不要求具体值，但关注趋势）
- **BDI**：应该保持在一个稳定范围（太高=散乱，太低=趋同）
- **RCD**：不应持续增长（级联风险）
- **SDE**：应接近 1.0（衰减模型有效）

### 停止条件
- ē_W 连续 3 个周期 < -0.5 → 暂停，通知人类（估计器严重偏移）
- 任何指标出现跳变（Δ > 2σ）→ 记录，但不自动行动

## 四、约束

1. **一次只改一个变量**——同 autoresearch 的"Agent 只改 train.py"。同时改三个钢印，无法归因效果
2. **每个实验至少跑 1 个完整裁决周期（6h）**——制度空间的反馈是延迟的
3. **不触碰红线目录**（同 MEMORY.md 系统红线）：
   - C:\Users\Administrator\.openclaw\
   - D:\npm\
   - C:\Users\Administrator\.hermes\
   - C:\Users\Administrator\.ssh\
   - E:\obsidian-vault\
4. **不修改 DENSITY-001 的采集系统**——那是"prepare.py"，只读
5. **不删除数据**——所有实验记录永久保留，失败的实验也有价值
6. **简单性优先**——同 autoresearch 的 Simplicity criterion。一个钢印如果 30 天未触发，与其增加复杂条件不如直接删除

## 五、研究日志格式

每次实验记录到 `research-log.jsonl`：

```json
{
  "id": "R001",
  "timestamp": "2026-05-24T09:00:00+08:00",
  "hypothesis": "将 D047 阈值从 0.90 降到 0.85 后，后悔率 r_t 会下降",
  "change": "CLAUDE.md: 修改 D047 阈值为 0.85",
  "metric_before": {"e_w": -0.36, "r_t": 0.0, "BDI": 1.2},
  "metric_after": {"e_w": -0.28, "r_t": 0.0, "BDI": 1.3},
  "result": "neutral",
  "kept": true,
  "note": "ē_W 改善但 r_t 无变化，可能需要更长观察期"
}
```

## 六、与 autoresearch 的关键差异

| 维度 | autoresearch | 端点星自适应研究 |
|------|-------------|---------------|
| 搜索空间 | 模型参数 | 行为制度 |
| 实验周期 | 5 分钟 | 6 小时 |
| 每天实验数 | ~100 | ~4 |
| 反馈信号 | val_bpb（密集、客观） | ē_W/BDI/RCD/SDE（稀疏、噪声） |
| 人类参与 | 睡觉，醒来收结果 | 睡觉，飞书收日报 |
| 风险 | 跑崩了就 discard | 改坏了可能影响系统稳定性 |
| 价值 | 找到更好的模型 | 找到更好的自我管理策略 |

**核心差异**：autoresearch 的失败成本几乎为零（discard 即可），端点星的失败成本是系统退化。因此端点星需要更保守的策略——每个改动都要有回滚能力。

## 七、启动检查清单

端点星开始自适应研究前，确认以下条件：

- [ ] research-log.jsonl 已创建
- [ ] 能读取 DENSITY-001 的最新裁决数据（HTTP 或本地）
- [ ] CLAUDE.md 当前版本已备份到 CLAUDE.md.baseline
- [ ] 理解五个指标的含义和判定逻辑
- [ ] 已阅读 Karpathy autoresearch 的 program.md（理解方法论内核）

## 八、从哪里开始？

推荐的前三个实验（从低风险到高风险）：

1. **R001：观察实验** — 不改任何东西，连续 3 个周期只记录指标，建立 baseline
2. **R002：清理未触发钢印** — 找到 30 天未触发的钢印，标记为候选删除，观察 BDI 变化
3. **R003：阈值微调** — 将最活跃的钢印阈值微调 ±0.05，观察 ē_W 变化
