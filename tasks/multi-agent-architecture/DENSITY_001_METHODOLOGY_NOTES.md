## 一、Karpathy 自动科研四层架构

Karpathy 描述的 AI 自动科研闭环：

```
Human researchers → 定义研究目标
Agent swarm      → 生成数千实验
Compute cluster  → 运行训练
Evaluation       → 自动筛选改进
```

## 二、端点星自进化系统的对应关系

| Karpathy 层 | 端点星对应 | 优化对象 |
|------------|-----------|---------|
| Human 定义目标 | 雪飞下指令、设决策编号 | — |
| Agent swarm 生成实验 | 每日复盘 + 月度进化自动运行 | 行为模组配置 |
| Compute 运行训练 | cron 子 agent 执行任务 | 实际操作 |
| Evaluation 筛选改进 | D048 效果衰减追踪（effectiveness_score） | 模组升降级 |

**关键差异**：Karpathy 的系统优化模型权重（参数空间），端点星优化行为制度（制度空间）。

## 三、制度空间搜索 vs 参数空间搜索

制度空间搜索更难，因为：

1. **反馈信号稀疏** — 模型训练有 loss function，钢印效果只能数"阻止了几次事故"
2. **延迟更长** — 一条钢印可能 30 天才触发一次，不是秒级反馈
3. **无 ground truth** — "这条钢印该不该降级"没有标准答案

这也解释了 D047 的设计原则"阈值由数据涌现"——在稀疏信号环境下，先验设定阈值几乎必然出错。

## 四、DENSITY-001 的方法论定位

DENSITY-001 不是 Karpathy 式的参数搜索，而是**给制度搜索装仪表盘**：

- Karpathy：在 GPU 上调参，loss function 明确
- 端点星：在活的运维系统里做控制变量实验
- DENSITY-001：用 ρ 量化"这个系统在变聪明还是变笨"

当 EXP-003（钢印效果量化）跑出数据，端点星可以输出一个 Karpathy 式指标——**钢印有效率**。当这个数字有趋势（上升/下降），就知道系统的"免疫力"在增强还是减弱。

## 五、对实验设计的启示

### 5.1 追踪颗粒度升级需求

Karpathy 的实验有明确的 loss curve，端点星需要等价的指标体系：

| 层次 | Karpathy 指标 | 端点星等价指标 | 当前状态 |
|------|-------------|--------------|---------|
| 实验吞吐 | experiments/min | 裁决轮次/天 | ✅ 已有 |
| 信号质量 | loss/reward | ρ̄ 滑动均值 | ✅ 已有 |
| 评估精度 | validation accuracy | 残差 ē_W | ✅ 已有 |
| 系统健康 | GPU utilization | 端点星可达率 | ⚠️ 粗粒度 |
| 行为多样性 | exploration entropy | 行为类型分布 | ❌ 缺失 |
| 因果链深度 | gradient path length | 引用链深度分布 | ❌ 缺失 |
| 信息新鲜度 | data staleness | S 衰减有效性 | ❌ 缺失 |

### 5.2 新增观测维度

基于方法论对比，新增三个核心观测维度：

1. **行为多样性指数（BDI）**：每轮裁决中不同 action_type 的 Shannon 熵
   - BDI = H(action_types) = -Σ p_i · log2(p_i)
   - BDI 下降 → 系统行为趋同化 → 可能陷入局部最优

2. **引用链深度（RCD）**：信息块引用链的最大深度
   - RCD = max(path_length from any leaf to referenced root)
   - RCD 增长 → 信息依赖链加长 → 级联失效风险上升
   - RCD 趋零 → 信息孤立 → 系统碎片化

3. **S 衰减有效性（SDE）**：λ 衰减后信息的实际引用率
   - SDE = P(referenced | λ < 0.5) / P(referenced | λ ≥ 0.5)
   - SDE ≈ 1.0 → 衰减模型正确，旧信息确实没用了
   - SDE >> 1.0 → 衰减过激，删了不该删的
   - SDE << 1.0 → 衰减不足，旧信息占着空间

## 六、ρ̂ 趋势：系统的体温计

Ridge 校正后的 ρ̂ = ρ + ê 是端点星信息密度的校准估计。

**关键洞察**：ρ̂ 趋势比 ρ̂ 单次值更重要。

- ρ̂ 稳定在某个范围 → 系统处于信息密度平台期
- ρ̂ 持续下降 → 系统在退化，需要干预
- ρ̂ 持续上升 → 系统在进化，但可能膨胀

这与 Karpathy 的 loss curve 类比：他不看单次 loss 值，看 loss 的趋势。我们的 ρ̂ 趋势就是端点星的 loss curve。

**什么时候可以说 DENSITY-001 产生了真正的科学发现？** 当 ρ̂ 趋势的斜率在统计上显著不为零，且被跨 agent 泛化验证确认。

## 七、经验教训

1. **对比是最好的镜鉴** — Karpathy 的框架帮我们发现了三个盲区。没有外部参照，系统会对自己缺少什么毫不知情
2. **指标不是越多越好** — 新增指标都有明确的判定逻辑和异常信号，不是为了全面而是覆盖已知盲区
3. **"阈值由数据涌现"不仅适用于 θ** — 三个新指标的异常阈值也应该从数据中涌现，不预设
4. **最难的问题最有价值** — 制度空间搜索比参数空间搜索难，但方法论价值也更大
5. **闭环不一定需要 GPU** — 端点星的实验在 CPU 上运行，证明自动科研不一定需要大算力

## 八、开源定位

本实验的核心贡献不是一个模型或一个工具，而是一套**可复现的实验方法论**：

- 如何给长期运行的 AI agent 装信息密度仪表盘
- 如何用双轨并行避免先验框架偏差
- 如何在稀疏信号、长延迟、无 ground truth 的约束下做科学实验

适用于任何需要量化评估 AI agent 长期运行质量的场景。
