# 经验教训

> 信息存活价值评估:LLM 微调 vs 回归 vs 混合,完整实验得出的教训。

## 全程轨迹

| 实验 | ρ | 说明 |
|---|---|---|
| LLM v1 (原始, 分2占53%) | nan | 坍缩,全输出 2 |
| LLM v2 (平衡① 分2→35) | 0.325 | 偏向新的多数类 4 |
| LLM v3 (彻底平衡 各类~28) | 0.159 | 压多数类过头,最大类判断崩 |
| **LLM 终版** (B标签) | **0.466** | 换标签,信号回到文本 |
| **回归 Ridge** | **0.778** | TF-IDF,同文本,碾压 |
| 回归 SVR | 0.779 | |
| Ensemble 等权 | 0.653 | 被 LLM 拖累 |
| Ensemble 0.7/0.3 | 0.688 | 仍低于回归 |

## 教训 1:小数据下简单模型碾压大模型

**现象**: 300 条训练,TF-IDF + Ridge (ρ=0.778) 远超 3B LLM QLoRA (0.466)。

**根因**:
1. 数据量太小——3B 模型参数远多于样本,学不过线性回归。
2. 任务线性可分——内容质量评分跟词频/短语强相关(「核心架构」vs「日报流水」),TF-IDF 直接抓线性特征,LLM 的语义优势用不上。
3. 杀鸡用牛刀——3B 学 1–5 序数评分,复杂度严重过剩。

**对策**: 小数据(<500)+文本任务,先跑 TF-IDF + Ridge/Logistic,大概率够用。别上来就 LLM。

## 教训 2:类别不平衡能让 LLM 坍缩

**现象**: 分2 占 53% 时,LLM 三轮都坍缩/偏向(v1 全输出2 ρ=nan,v2 偏4,v3 压过头 ρ=0.159)。

**根因**: 标准交叉熵下,多数类主导梯度,模型发现「无脑答多数类」就能拿高准确率,于是学这个捷径,不学特征。

**反直觉陷阱**: 把多数类压太狠(v3 把分2 从 103 压到 28)反而伤了测试集中该最大类的判断。平衡要适度,优先补真实少数类样本或加权 loss,而非暴力欠采样。

**对策**: 用 `split.py` 的分层切分 + CAP 上限平衡(CAP=80,多数类压到上限,少数类全保留,不过采样)。

## 教训 3:混合模型绝不是简单 ensemble

**现象**: LLM 和回归在 61 条测试上只有 3 条一致(5%),分歧 58 条。分歧中 LLM 更近人工 34 条 > 回归 24 条——LLM 有局部校准价值。但简单 ensemble(等权 0.653、加权 0.688)都低于回归单模型 0.778。

**根因**: LLM 整体弱(ρ低),简单平均会被弱模型拖累。LLM 的局部优势(34 条更准)被全局平均稀释。

**对策**: 要让 LLM 只在它强的地方介入:
- **置信度门控**: 回归高置信用回归,低置信(预测落在中间带)时参考 LLM
- **stacking**: 训练元模型决定何时信哪个(但元模型也要数据)
- **边界校准**: LLM 只校准回归预测落在 3/4 或极值附近的 case

千万别 `0.5*reg + 0.5*llm` 了事。

## 教训 4(最重要):先算简单基线,再上大模型

整个项目绕了一大圈(标签选错→换标签→平衡调参→三轮训练→才回头算回归),最后发现 TF-IDF + Ridge 一直就在终点(0.778)。**第一轮就该先花 5 秒跑回归**,能省掉几小时 LLM 调参弯路。

这是超越具体任务的方法论:**任何建模任务,先跑最简单的基线**。

## 教训 5:标签定义决定模型选择

「存活价值」度量什么?
- **被调用频次**(信号在 metadata:引用次数/目录/更新频率)→ 回归天然占优,LLM 读全文看不到
- **内容质量**(信号在文本)→ LLM 有牌打

本项目第一轮用「频次」标签失败(LLM 读不到 metadata),换「质量」标签后 LLM 从 0.16→0.466。**动手前先想清楚标签依赖什么信号**,这决定该用 LLM 还是结构化模型。

## 工程坑(Blackwell/Windows)

1. 5060 Ti 是 Blackwell(sm_120),torch **必须 cu128**,cu126 报 no kernel image
2. Python 3.14 + cu128 wheel 已可用(torch 2.11.0+cu128 cp314),不必降级 3.11
3. pip 在某些机器下载极慢(~150kB/s)但 curl 同网 17MB/s——大 wheel 用 curl 下完 `pip install xxx.whl` 离线装
4. Windows GBK 终端 print `✓`/`ρ` 会 UnicodeEncodeError——脚本开头加 `sys.stdout.reconfigure(encoding='utf-8', errors='replace')`(否则训练成功了也崩在最后 print,误判失败)
5. trl 1.x API:`SFTTrainer(tokenizer=)` → `processing_class=`;`SFTConfig(max_seq_length=)` → `max_length=`;`dataset_text_field` 仍可用
6. HF 模型下载国内用镜像:`HF_ENDPOINT=https://hf-mirror.com`
